How to import transactions via CSV

Return to index

Building the system (app) using Import transactions (csv) is the quickest method of creating
transactions so that the 24 Month Projection can be populated.
It relies on you creating a spreadsheet containing a row for each transaction. The spreadsheet
should then be saved as a CSV file to a location on your Computer or Tablet that is readily
accessible for any subsequent re-loads.
The format of the spreadsheet is strict and must adhere to the following specification;

The data should be in the format;
Column A:Header text=Date, content - standard date format
Column B:Header text=Amount, content - numeric
Column C:Header text=Income/Expense, content - either the text income or expense
Column D:Header text=Category , content - Text containing your chosen category
Column E:Header text=Description , content - Text containing a description of the transaction.
Column F:Header text=Frequency , content - either the text Monthly or 4-Weekly
Note! 
The header row is case sensitive and must be as shown.
The Income/Expense data is also case sensitive, lower case.
The frequency is case sensitive and must be as shown.

On the first build of the system, enter start date and opening balance, then click on 
choose file next to Import transactions CSV  then click Import (replace existing)
The transactions will instantly load. If there are any errors and alert will inform you.
Simply scroll to see all your transactions.

An example of loading data
If you have a start date of 12th Jan 2026 with an opening balance of £x.yz
your spreadsheet should have, as it's initial transaction, one dated 12th Jan 2026 (if you have one then)
OR one for the NEXT date after that. Then add your transactions in date order from then till
11th Feb 2026. This gives you one month's worth of transactions to load into the app.

How to edit transactions

Return to index

To Edit a transaction, click on View Transactions
Find the transaction to be edited and click Edit
The date is loaded into the input boxes above the transaction table.
Edit the data as required and click Save
The changes will then be reflected within the transaction table and also in the 
24 Month Projection table balances.

How to delete transactions

Return to index

To Delete a transaction, click on View Transactions

Find the transaction to be deleted and click Delete
You will then be prompted to either carry on and delete it or cancel
If you delete, the transaction will be removed from the Transaction table
and from the 24 Month Projection table.

Negative Balances

Return to index

The Negative Balances button will display all dates in the 24 Month Projection
table with balances less than zero in a popup display similar to the Salary -1 day popup.
If there are no negative balances, an alert will display a message.
if there are negative balances, they will be displayed. 
You can click on any of these and it will take you to the transaction.

Salary-1

Return to index

The Salary -1 day balances button, above 24 Month Projection table displays your account balance 
on the day before each Salary. As this is the most likely time your account may  be
negative this may be a useful feature.

In the popup, scroll to see the various balances, if you do find a negative balance, click it
and it will take you to the relevant transaction.
