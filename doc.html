Diary

Return to index

Diary allows you to enter notes against past, present and future dates to record
information you feel is related to the relevant transaction.
eg. You may want to note a future change where the transaction amount changes on a certain date.

The records may be searched.

Future transactions will flag up before they are due to allow you to make the necessary changes.

You can also click the Next 14 days Diary button which will show you all Diary
entries for the next 14 days.
The Diary may be backed up, and restored if required.

Whatif

Return to index

You can have a What if scenario and use the 24 Month Projection table to see
the effect of adding either an expense or income.

The most likely scenario would be adding an expense to see what the effect is on your
future balances.

For example
You want to see the effect of saving ¬£100 per month for Holidays
Add a transaction, enter Holiday Savings against the Description, enter ¬£100 
for the amount, select a date and Category for it and Add Transaction

The 24 month projection is immediately updated.
You can either scroll down the table to see the effect, or a quicker way is to look at
Negative balances or Salary -1 balances

Once you see the effect you can keep the transaction, or if the expense is too much
either delete it or reduce the amount, depending on your circumstances or preference.

How to set up manually

Return to index

First, enter a start date by clicking into Start date field and selecting a date from the date picker.
Next, enter the Opening Balance of your bank account on that date, then click Save.
Note!if your bank balance on the start date is (eg) ¬£150, and you have transactions 
on the same date, add the transactions to your bank balance and this should be your opening
balance. eg. if your transactions are ¬£50, bank balance ¬£150, opening balance should be ¬£200


You should now add your Categories. eg. Household, Insurance, Savings etc. Enter your first Category in 
the box above Transaction table, then click Add Category. Add as many Categories as you need, 
you can add more as required. 
Now start adding your transactions by adding a calendar month's worth to ensure you 
have all your data entered.

Enter the Description, next the Amount, then select whether the 
transaction is Expense or Income. 

The next field allows to select the transaction Period
normally either Monthly or, if it's relevant 4-Weekly. 
You can select Irregular for one off transactions. 

The next field allows you to select the Date for the transaction - either the date for 
Monthly transactions, or the first date after the Start date of a 4-weekly transaction
Next select the appropriate Category for the transaction. 

Finally, click Add Transaction.
The transaction will appear in the table below. 
Add as many transactions as you require.

As you add transactions, they will appear in the 24 Month Transactiontable.

Note!
On first use, you should enter, starting from your  Start date all of your Monthly transactions 
for one calendar month. If you have 4-Weekly transactions add the next date after the start date they 
are due and ensure you enter all of this type.

The system will calculate the transactions and populate the 24 Month Projection table.

If you do have 4-Weekly transactions, once the date you enter initially has passed, the system will 
alter the day of the month shown in the transaction table by adding 28 days to ensure 
these days are always accurate

How Data is stored

Return to index

When you enter a start date and opening balance then either load transactions
manually or via CSV loader, or via Import transactions JSOB, the data is stored using local storage.
local storage is a type of web storage that allows a website or web application to 
store data directly in the user's browser. It acts like a small, private filing cabinet for 
that specific website on your computer or tablet.

‚Ä¢ Persistent: Unlike Session storage (which disappears when you close the tab), data in local storage 
stays there even if you close the browser or restart your device.¬† 
‚Ä¢ Domain-Specific: Data saved by website-a.com cannot be read by website-b.com. 
This ensures privacy and security between different apps.
‚Ä¢ It is also browser specific.
‚Ä¢ No Expiration: The data remains until the web app deletes it or you manually 
clear your browser's cache/cookies
‚Ä¢ Note! If you do clear the browser cache for any reason, you can easily re-build
you app by entering a relevant start date and opening balance and importing your transactions
using CSV import - this takes less than a minute to do.

How to import transactions via CSV

Return to index

Building the system (app) using Import transactions (csv) is the quickest method of creating
transactions so that the 24 Month Projection can be populated.
It relies on you creating a spreadsheet containing a row for each transaction. The spreadsheet
should then be saved as a CSV file to a location on your Computer or Tablet that is readily
accessible for any subsequent re-loads.
The format of the spreadsheet is strict and must adhere to the following specification;

The data should be in the format;
Column A:Header text=Date, content - standard date format
Column B:Header text=Amount, content - numeric
Column C:Header text=Income/Expense, content - either the text income or expense
Column D:Header text=Category , content - Text containing your chosen category
Column E:Header text=Description , content - Text containing a description of the transaction.
Column F:Header text=Frequency , content - either the text Monthly or 4-Weekly
Note! 
The header row is case sensitive and must be as shown.
The Income/Expense data is also case sensitive, lower case.
The frequency is case sensitive and must be as shown.

On the first build of the system, enter start date and opening balance, then click on 
choose file next to Import transactions CSV  then click Import (replace existing)
The transactions will instantly load. If there are any errors and alert will inform you.
Simply scroll to see all your transactions.

An example of loading data
If you have a start date of 12th Jan 2026 with an opening balance of ¬£x.yz
your spreadsheet should have, as it's initial transaction, one dated 12th Jan 2026 (if you have one then)
OR one for the NEXT date after that. Then add your transactions in date order from then till
11th Feb 2026. This gives you one month's worth of transactions to load into the app.

Find the transaction to be deleted and click Delete
